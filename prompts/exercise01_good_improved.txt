# 用語の定義（重要・解釈を統一すること）

集計のブレを防ぐため、以下の定義を必ず守ってください。

- 期限切れタスク … 「期限日 < 今日」かつ「未完了」のタスク
- 今日のタスク   … 「期限日 = 今日」かつ「未完了」のタスク
- 今週のタスク   … 「期限日が今週（月曜始まり〜日曜まで）」かつ「未完了」のタスク
                   ※ 今日のタスクも今週内であれば、今週のタスク数に含める（重複してよい）
- 進捗率(%)      … 完了タスク数 ÷ 総タスク数 × 100（小数点以下は四捨五入）
                   ※ 総タスク数が0件のときは 0% と表示する

# データモデル

各タスクは以下の情報を持つこと。

- id（一意なID）
- title（タイトル・必須）
- description（説明・任意）
- createdAt（作成日・自動付与）
- dueDate（期限日）
- status（"未完了" または "完了"）

# 必須機能

## タスク操作
- 追加 / 編集 / 削除 / 完了にする / 未完了に戻す

## 編集UI
- 編集ボタンを押すと、入力欄に既存の値が入った状態で編集できること
  （実装方法はモーダル・インラインのどちらでもよいが、1つに統一すること）

## フィルター（タスク一覧に適用）
- すべて / 未完了 / 完了済み / 期限切れ

## 一括操作
- すべて完了にする / 完了済みタスクを一括削除

## データ保存
- localStorage に保存し、ブラウザを再起動してもタスクが残ること
- 起動直後が空で寂しくならないよう、初回起動時のみサンプルタスクを3件投入すること
  （期限切れ1件・今日1件・今週内の未来日1件が含まれるようにする）

1. ヘッダー … アプリタイトル＋簡単な説明文

- 棒グラフは使わない（種類を統一して再現性を保つ）。
- 外部ライブラリ・CDNの使用は禁止。

- 使用技術は HTML / CSS / JavaScript のみ。
- 外部ライブラリ・CDN・フレームワーク・サーバーは禁止。

# コーディングルール

- HTML / CSS / JavaScript をファイル分離する。
- 主要な画面枠と入力フォームはHTMLに静的に記述し、タスク一覧の各項目はJavaScriptで動的に生成する。
- 1つの関数に1つの責務だけ持たせる（集計・描画・保存・イベント処理を混在させない）。
- 分かりやすい変数名・関数名を使う。
- 日本語コメントで「何のための処理か」を説明する。
- 重複コード・重複ロジックを避ける（共通処理は関数化する）。

# 表示・非表示の切り替えルール（不具合防止・必ず守ること）

モーダルや空メッセージなど「出したり隠したりする要素」で、表示切り替えが効かなくなる不具合を防ぐため、以下を必ず守ること。

- 要素の表示・非表示を切り替える方法を1つに統一すること（例：`hidden` 属性で統一する、もしくは表示用のクラスの付け外しで統一する）。両者を混在させない。
- HTMLの `hidden` 属性で隠す要素には、CSSで `display` を無条件指定しないこと。
  - 理由：`hidden` 属性は内部的に `display:none` で隠す仕組みのため、`.modal { display:flex }` のように常時 `display` を指定すると `hidden` が打ち消され、隠れなくなる（＝モーダルが常に画面を覆い、操作不能になる）。
  - 正しい例：表示時のスタイルは `.modal:not([hidden]) { display:flex; }` のように「隠れていないときだけ」適用する。隠す指定（`display:none`）は別途書かない。
- 初期状態（ページ読み込み直後）で隠しておきたい要素は、HTML側で最初から `hidden` を付けておくこと。
- モーダルを使う場合は「開く」「閉じる」を担当する関数を分け、閉じる手段（キャンセルボタン・背景クリック等）からも確実に閉じられること。
